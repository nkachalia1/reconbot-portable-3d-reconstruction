# Evaluation Plan

The project should produce quantitative artifacts for every dataset. Store them under `outputs/<session>/`.

## Calibration Metrics

- Overall RMS reprojection error in pixels.
- Per-image reprojection error distribution.
- Number of accepted and rejected checkerboard images.

Target:

- Under 1.0 px is good for a webcam.
- Under 0.5 px is excellent if the checkerboard images are sharp.

## Feature Metrics

Generated by:

```powershell
reconbot feature-benchmark --images data/frames/session_001 --output outputs/session_001/features.csv
```

Track:

- SIFT vs ORB keypoints.
- Adjacent-frame match count.
- RANSAC fundamental-matrix inlier ratio.
- Runtime per pair.

Use in interview:

- SIFT is often more robust.
- ORB is faster and Pi-friendly.
- Inlier ratio is a practical proxy for match precision when there is no manual ground truth.

## SfM Metrics

Generated by:

```powershell
reconbot pair-sfm --image-a ... --image-b ... --intrinsics configs/camera_intrinsics.yaml --output outputs/session_001/pair_000
```

Track:

- Raw matches.
- Essential matrix inliers.
- Triangulated 3D points.
- Mean, median, RMSE, and max reprojection error.

For full COLMAP models, also track:

- Camera count. A fixed monocular camera should normally produce one shared
  camera model.
- Registered-image count and percentage.
- 3D point and observation counts.
- Mean track length.
- Mean observations per image.
- Mean reprojection error.

Do not optimize for reprojection error alone. A model with one camera per image
can report a lower error by overfitting intrinsics while producing fewer,
less-constrained 3D points.

## Point Cloud Metrics

Generated by:

```powershell
reconbot pointcloud-metrics --ply outputs/session_001/pair_000/points.ply --voxel-size 0.05 --output outputs/session_001/pair_000/cloud_metrics.json
```

Track:

- Number of points.
- Axis-aligned bounding box.
- Points per cubic meter.
- Occupied voxel count.
- Voxel coverage ratio.

Note: monocular SfM has arbitrary scale unless you add a known-size calibration object or measured baseline.

## Recommended Experiment Table

| Experiment | Variable | Expected Tradeoff |
| --- | --- | --- |
| Slow vs fast capture | motion speed | fast capture increases blur and lowers inlier ratio |
| ORB vs SIFT | feature type | ORB is faster, SIFT is usually more robust |
| Sparse vs dense texture | scene texture | low texture reduces points and completeness |
| Short vs wide baseline | camera motion | short baseline hurts depth, wide baseline hurts matching |
| Pi vs laptop | compute platform | Pi is field-friendly, laptop is reconstruction-friendly |
